§long_lived_resources: SlotMap<InnerHandle, (GpuRes, Arc<InnerHandle>)>

We store a refcounted handle along side every long lived resource so we can tell if it is still alive. This mechanism is similar to crate::wgpu_resources::DynamicResourcePool, only that we don’t retain for another frame.

Perf note: If we get a lot of resources this might scale poorly (well, linearly) If this happens, we need to make the handle more complex and give it an Arc<Mutex<>> of a free list where it can enter itself. (which makes passing the handles & having many short lived handles more costly)
