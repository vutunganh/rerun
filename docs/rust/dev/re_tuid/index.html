The default string format is big-endian hex, e.g. 182342300C5F8C327a7b4a6e5a379ac4. This means the string representation sorts the same.

It is common to prefix an TUID with a namespace. This is done as: {namespace}_{tuid} where namespace can be anything but is recommended to be:

  • ASCII

The idiomatic way of implementing this is to wrap Tuid in a newtype struct (e.g. struct UserId(Tuid)) and implement the prefix there.

It is recommended that

  • Finding the wrong prefix is an error
  • A missing prefix is NOT an error
